# Everway Design System - Mandatory Usage Rules

This project uses the **Everway Design System**. All UI code MUST use design system components and tokens. Prefer inline utility classes, BEM classes, and token values over hardcoded magic numbers/values where available.

## 🚨 CRITICAL SETUP: Theme Attribute Required

**Before ANY Everway styles will work, you MUST set the theme attribute on the root element:**

```html
<html lang="en" data-eds-theme="light">
```

Or in React/Next.js:

```tsx
// In your root layout or App component:
<div data-eds-theme="light">
  {children}
</div>

// Next.js app/layout.tsx:
export default function RootLayout({ children }) {
  return (
    <html lang="en" data-eds-theme="light">
      <body>{children}</body>
    </html>
  );
}
```

**⚠️ Without `data-eds-theme`, NO CSS variables are defined and all components will be unstyled!**

Valid values: `"light"` or `"dark"`

For complete setup instructions, use: `mcp_everway-design-system_get_setup`

## Component Mapping - NEVER Build Custom

### ❌ NEVER DO THIS:

```tsx
// WRONG: Custom button with inline styles
<button style={{ backgroundColor: 'blue', padding: '10px' }}>Click</button>

// WRONG: Custom button with arbitrary classes
<button className="my-custom-btn bg-blue-500">Click</button>

// WRONG: Using raw Radix primitives directly
import * as Dialog from '@radix-ui/react-dialog';
<Dialog.Root>...</Dialog.Root>

// WRONG: Using shadcn/Chakra/MUI/Ant directly
import { Button } from "@/components/ui/button";  // shadcn
import { Modal } from "@chakra-ui/react";
import { DataGrid } from "@mui/x-data-grid";

// WRONG: Custom classes for component structure (use subcomponents!)
<Card>
  <div className="card-header">...</div>   {/* ❌ Use Card.Header */}
  <div className="card-body">...</div>     {/* ❌ Use Card.Body */}
  <div className="card-footer">...</div>   {/* ❌ Use Card.Footer */}
</Card>

// WRONG: Custom icon/avatar classes (use Thumbnail!)
<span className="product-icon" />          {/* ❌ Use Thumbnail */}
<div className="user-avatar" />            {/* ❌ Use Thumbnail */}
```

### ✅ ALWAYS DO THIS:

```tsx
// CORRECT: Use Everway components
import { Button, Card, Dialog, Thumbnail, SmartTable } from "@everway/design-system-react";

<Button variant="primary" size="md">Click</Button>

// CORRECT: Use Dialog with all subcomponents
<Dialog>
  <Dialog.Trigger asChild>
    <Button>Open</Button>
  </Dialog.Trigger>
  <Dialog.Content>
    <Dialog.Title>Title</Dialog.Title>
    <Dialog.Description>Content here</Dialog.Description>
  </Dialog.Content>
</Dialog>

// CORRECT: Use Card with subcomponents (not custom classes!)
<Card>
  <Card.Header>
    <Thumbnail size="sm" color="purple">
      <ProductIcon />
    </Thumbnail>
    <span style={{ font: 'var(--eds-title-lg)' }}>Product Name</span>
  </Card.Header>
  <Card.Body>Main content</Card.Body>
  <Card.Footer>
    <Button variant="tertiary">View Details</Button>
  </Card.Footer>
</Card>
```

**FormField has NO subcomponents.** It's just a layout wrapper `<div>`. Use these standalone components and CSS classes:

### Colors

```scss

// ✅ CORRECT
color: var(--eds-color-text-primary);
```

| Tailwind | Everway      | Value |
| -------- | ------------ | ----- |
| `p-2`    | `eds-p-xs`   | 8px   |
| `p-3`    | `eds-p-md`   | 12px  |

### Typography

Typography tokens are CSS shorthand values (weight size/line-height font-family).

**Display** (hero/marketing): `display-xl`, `display-lg`, `display-md`, `display-sm`
**Headline** (page/section headings): `headline-xl`, `headline-lg`, `headline-md`, `headline-sm`
**Title** (component titles): `title-xl`, `title-lg`, `title-default`, `title-sm` (+ `-strong` variants)
**Body** (paragraphs/content): `body-lg`, `body-default`, `body-sm`

```scss
// ❌ WRONG
font-size: 14px;
font-weight: bold;

// ✅ CORRECT
font: var(--eds-headline-lg);
font: var(--eds-title-default);
font: var(--eds-body-default);
font: var(--eds-title-sm-strong); // for emphasis
```

### Border Radius

| Tailwind       | Everway           | Value |
| -------------- | ----------------- | ----- |
| `rounded`      | `eds-radius-xs`   | 8px   |
| `rounded-lg`   | `eds-radius-md`   | 16px  |
| `rounded-full` | `eds-radius-pill` | 112px |

## CSS Class Naming Convention

All Everway classes use the `eds-` prefix:

-   Base class: `eds-{component}` (e.g., `eds-button`, `eds-card`)
-   Variants: `eds-{component}--{variant}` (e.g., `eds-button--primary`)
-   States: `eds-{component}--{state}` (e.g., `eds-button--disabled`)
-   Utilities: `eds-flex`, `eds-align-center`, `eds-gap-xl`, etc.

## Import Path

ALL design system imports MUST come from:

```tsx
import { ComponentName } from "@everway/design-system-react";
```

## Workflow Checklist

When asked to build UI, follow this EXACT process:

1. [ ] Call `list_components` to review available components
2. [ ] For common patterns, call `get_use_case` or `get_layout` first
3. [ ] For each needed element, call `get_component` for details
4. [ ] **Check subcomponents** - use `Component.Subcomponent` (e.g., `Card.Header`) instead of custom classes
5. [ ] Use `generate_code` to get correct implementation
6. [ ] If migrating from another library, use `translate_component`
7. [ ] Only use `eds-` CSS classes and `var(--eds-*)` tokens
8. [ ] Import all components from `@everway/design-system-react`

### Self-Check Before Creating Custom CSS

Before writing any custom CSS class (e.g., `.my-header`, `.product-icon`), ask:

-   [ ] Does this component have a subcomponent for this? (Check table above)
-   [ ] Can I use an existing EDS component instead? (e.g., `Thumbnail` for icons)
-   [ ] If styling is needed, am I preferring utility classes, BEM classes, and EDS token values over hardcoded values?

If you answered NO to the first two, you may create a custom class and should use EDS tokens instead of hardcoded values.

## Exceptions

Custom styling is generally reserved for these cases:

1. No design system component exists (verify with `search` tool first)
2. Explicitly approved by the design system team
3. Layout utilities are needed - use `translate_tailwind` for guidance

Even in exceptions, you MUST still use design system tokens for colors, spacing, and typography.

## Code Generation Rules

1. **Before generating any JSX/TSX:** Call the relevant `get_component` MCP tool
2. **For full pages:** Call `get_use_case` or `get_layout` first
3. **Before suggesting colors:** Call `get_tokens("colors")`
4. **Before suggesting spacing:** Call `get_tokens("spacing")`
5. **Before suggesting typography:** Call `get_tokens("typography")`
6. **When user pastes Tailwind classes:** Call `translate_tailwind` to convert
7. **When user mentions Radix:** Call `translate_radix` to find the wrapper
8. **When user mentions another library:** Call `translate_component`

## Response Format for UI Tasks

When implementing UI, structure your response as:

1. **Design System Query:** "Let me check the design system for the right component..."
2. **Component Selection:** "The Everway Design System provides [Component] for this..."

## Red Flags - Avoid By Default

These patterns usually indicate a design system violation when utility classes, BEM classes, or token values are available:

```tsx
// ❌ Hardcoded inline style values
style={{ color: 'blue' }}
style={{ padding: 20 }}
style={{ backgroundColor: '#fff' }}

// ❌ Arbitrary class names
className="custom-button"
className="my-modal"
className="btn-primary"  // unless it's eds-button--primary

// ❌ Raw HTML for interactive elements
<button>  // should be <Button>
<input>   // should be <Input>
<select>  // should be <Select>

// ❌ Direct Radix imports
import * as Dialog from '@radix-ui/react-dialog'
import { Popover } from '@radix-ui/react-popover'

// ❌ Other UI library imports
import { Button } from "@/components/ui/button"  // shadcn
import { Modal } from "@chakra-ui/react"
import { DataGrid } from "@mui/x-data-grid"
import { Collapse } from "antd"

// ❌ Hardcoded magic values in SCSS
color: #333;
padding: 16px;
font-size: 14px;
border-radius: 8px;
```

## Allowed Patterns

```tsx
// ✅ Design system components
import { Button, Dialog, Card } from "@everway/design-system-react";
<Button variant="primary" size="md">

// ✅ Design system classes
className="eds-button eds-button--primary"

// ✅ Design system utility classes
className="eds-flex eds-align-center eds-gap-xl"

// ✅ Design system tokens in SCSS
color: var(--eds-color-text-primary);
padding: var(--eds-spacing-md);
font: var(--eds-body-body-m);
border-radius: var(--eds-radius-md);

// ✅ Layout utilities (with design system spacing)
display: flex;
gap: var(--eds-spacing-md);

// ✅ Correct FormField pattern (no subcomponents!)
<FormField>
  <Label htmlFor="email">Email</Label>
  <Input id="email" type="email" />
  <p className="eds-form-label-alt">Helper text</p>
</FormField>
```

## Self-Check Before Submitting Code

Ask yourself:

1. Did I query the design system MCP before writing this UI?
2. Did I check for use cases or layouts that match this request?
3. Am I importing from `@everway/design-system-react`?
4. Do all my CSS classes start with `eds-`?
5. Do all my CSS values use `var(--eds-*)`?
6. Am I using semantic components instead of raw HTML?

If NO to any of these, revise the code.
